# Project Summary

- Introduced a metadata-driven adapter architecture aligned with official docs references for `claude`, `codex`, `qwen`, and `antigravity`.
- Upgraded canonical standards to include `skills`, `workflows`, `mcp`, and `agents` catalogs with richer production-oriented prompt content.
- Added schema validation gate before `init/link/sync/update` and improved backup handling for symlink-based installs.
- Enhanced CLI UX with branded banner, status icons, verbose/json doctor modes, and clearer diagnostics.
- Added keyboard-driven interactive wizard mode for no-argument launch with arrow/enter navigation and space-based agent multi-select, plus TTY fallback behavior for non-interactive environments.
- Improved interactive UX with centered fullscreen menu rendering, persistent post-action loop, and smoother return-to-menu flow.
- Hardened runtime path resolution by separating coordinator repo root from invocation cwd, enabling reliable adapter/standards loading when CLI is executed outside the repo directory.
- Extended `doctor` output contract with explicit `verification_level` (`verified|partial|assumed`) and `verification_source`, surfaced in both JSON and table summary outputs.
- Added `doctor --strict` policy gate to fail with non-zero exit code when any agent remains non-verified, enabling CI enforcement for compatibility confidence.
- Expanded compatibility matrix to include docs provenance, verification state, and asset-kind coverage.
- Published bilingual documentation: `README.md` (English) and `README.vi.md` (Vietnamese).
- Corrected Codex integration mapping to prompt-discoverable targets and improved interactive usability with per-line centered sub-screens plus explicit post-command keyboard actions (`Enter` return, `Q`/`Esc` exit).
- Added adapter-aware asset packaging pipeline across all supported agents with managed `ai-coordinator` target subfolders, plus Codex skill package generation (`SKILL.md` + `agents/openai.yaml`) for discoverable tool-native integration.
- Hardened migration from legacy root-level links by auto-repairing conflicting parent junctions before apply, preventing recursive nested-directory corruption during managed subfolder rollout.
